Añadiendo una nueva rama new-design

Actualización para añadir una nueva rama. A continuación se hará la fusión de la rama secundaria 'new-design' con la rama principal que en este caso es master.

Git Merge: Incorpora los cambios hechos en una rama sobre los hechos en otra rama. La rama _destino_ es la que acoge los cambios de la rama _origen_.
Inteligente: intenta hacerlo con el menor impacto sobre el repositorio posible. Nunca se perderá ninguna linea de código y ante la duda generará un _conflicto_.
Es el flujo de trabajo más recomendado para el día a día cuando uses git.

utilizando comandos git

git status : para ver el estado de los archivos, si hay modificaciones, la rama actual, los cambios, los archivos borrados y cambiados.
git diff : para ver que cambios exactamente se han realizado en los archivos. Edición y eliminación.
git add [nombre archivo] : para incorporar los cambios al repositorio. Añadirlo para el próximo commit.
git reset HEAD [nombre archivo] : para revertir los cambios del comando anterior.
git rm [nombre archivo] : para eliminar el archivo completamente del respositorio.
git commit , i para modo insert -> agregar el comentario, presionar ctrl + c -> :wq -> enter
git reset --soft HEAD~1 para restablecer el ultimo commit sin eliminar archivos.
git commit -m "COMENTARIO". para hacer el commit de manera más simple sin entrar al editor git push : para subir archivos al repositorio.
git remote : para ver la lista de repositorios relacionados.
git remote -v : para ver la url de los repositorios.
git push origin master : comando completo para hacer un push al repositorio.
git pull origin master : comando para descargar los cambios desde el repositorio.
git commit -am "COMENTARIO" : para añadir directamente todos los archivos al commit y el commentario.
